Para que tu proyecto **CarMeet** sea irreprochable y parezca escrito exactamente por la misma mano (tu profe), debemos aplicar sus patrones estructurales:

1. **Lombok intensivo** (`@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor`).
2. **Manejo de excepciones** usando `EntityNotFoundException` o `RuntimeException` capturadas por el `GlobalExceptionHandler`.
3. **Inyección de dependencias** exclusivamente a través de `@RequiredArgsConstructor`.
4. **Respuestas estandarizadas** envolviendo los retornos en tu `ApiResponse`.
5. **Código formateado verticalmente** (nada de variables o métodos en una sola línea).

Aquí tienes la **FASE 5** y **FASE 6** refactorizadas al estilo del profe, sin saltar nada.

```java
// MODEL
@Entity
@Table(name = "evento")
@Data @NoArgsConstructor @AllArgsConstructor @Builder
public class Evento {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String nombre;
    private String fecha;
    private String ubicacion;
}

// DTO
@Data @Builder
public class EventoDTO {
    @NotBlank(message = "El nombre es obligatorio")
    private String nombre;
    @NotBlank
    private String fecha;
    @NotBlank
    private String ubicacion;
}

// REPOSITORY
public interface EventoRepository extends JpaRepository<Evento, Long> {}

// SERVICE
@Service
@RequiredArgsConstructor
public class EventoService {

    private final EventoRepository repo;

    public Evento crear(EventoDTO dto) {
        Evento e = Evento.builder()
                .nombre(dto.getNombre())
                .fecha(dto.getFecha())
                .ubicacion(dto.getUbicacion())
                .build();
        return repo.save(e);
    }

    public List<Evento> listar() {
        return repo.findAll();
    }
}

// CONTROLLER
@RestController
@RequestMapping("/api/events")
@RequiredArgsConstructor
public class EventoController {

    private final EventoService service;

    @PostMapping
    @PreAuthorize("hasRole('ADMIN')")
    public ResponseEntity<ApiResponse<Evento>> crear(@Valid @RequestBody EventoDTO dto) {
        return ResponseEntity.ok(ApiResponse.<Evento>builder()
                .success(true)
                .data(service.crear(dto))
                .build());
    }

    @GetMapping
    @PreAuthorize("hasAnyRole('USER','ADMIN')")
    public ResponseEntity<ApiResponse<List<Evento>>> listar() {
        return ResponseEntity.ok(ApiResponse.<List<Evento>>builder()
                .success(true)
                .data(service.listar())
                .build());
    }
}
```

#### 3. ms-ticketing (Puerto 8093)

*Requiere WebClient Config.*

```java
// CONFIG
@Configuration
public class WebClientConfig {
    @Bean
    public WebClient webClient() {
        return WebClient.builder().build();
    }
}

// SERVICE
@Service
@RequiredArgsConstructor
public class TicketService {

    private final TicketRepository repo;
    private final WebClient webClient;

    public Ticket comprar(TicketDTO dto, String token) {
        
        Ticket t = Ticket.builder()
                .eventoId(dto.getEventoId())
                .precio(dto.getPrecio())
                .estado("PENDIENTE")
                .username(SecurityContextHolder.getContext().getAuthentication().getName())
                .build();

        t = repo.save(t);

        try {
            webClient.post()
                    .uri("http://localhost:8094/api/payments/procesar")
                    .header("Authorization", "Bearer " + token)
                    .bodyValue(t)
                    .retrieve()
                    .bodyToMono(Void.class)
                    .block();
            
            t.setEstado("PAGADO");
        } catch (Exception e) {
            t.setEstado("RECHAZADO");
        }

        return repo.save(t);
    }
}

// CONTROLLER
@RestController
@RequestMapping("/api/tickets")
@RequiredArgsConstructor
public class TicketController {

    private final TicketService service;

    @PostMapping("/comprar")
    @PreAuthorize("hasRole('USER')")
    public ResponseEntity<ApiResponse<Ticket>> comprar(
            @Valid @RequestBody TicketDTO dto, 
            @RequestHeader("Authorization") String authHeader) {
        
        // Limpiamos el Bearer para pasar solo el token al service
        String token = authHeader.substring(7); 
        
        return ResponseEntity.ok(ApiResponse.<Ticket>builder()
                .success(true)
                .data(service.comprar(dto, token))
                .build());
    }
}
```

```java
@Service
@RequiredArgsConstructor
public class RecintoService {

    private final RecintoRepository repo;

    public Recinto registrarIngreso(Long id) {
        Recinto r = repo.findById(id)
                .orElseThrow(() -> new EntityNotFoundException("Recinto no encontrado"));

        if (r.getOcupacionActual() >= r.getCapacidadMaxima()) {
            throw new RuntimeException("¡Recinto Lleno! Capacidad máxima alcanzada.");
        }

        r.setOcupacionActual(r.getOcupacionActual() + 1);
        return repo.save(r);
    }
}
```

### FASE 6: Simulacro de Defensa (Ejercicios y Soluciones)

Si el profesor te interroga sobre modificaciones rápidas, respóndele exactamente de esta forma técnica:

**Prueba 1: "Haz que la capacidad máxima de un recinto no pueda ser número negativo al crearlo."**

* **Solución:** Vas a `ms-venue-capacity`, paquete `dto`, clase `RecintoDTO`. Agregas `@Min(value = 1, message = "La capacidad debe ser mayor a 0")` sobre la variable `capacidadMaxima`. Guardas y reinicias.
* **Defensa:** *"Profesor, agregué la validación JSR-380 (`@Min`) en el DTO. Al fallar la validación, el `GlobalExceptionHandler` captura la excepción `MethodArgumentNotValidException` y devuelve nuestro objeto `ApiResponse` formateado en JSON con un código HTTP 400 (Bad Request)."*

**Prueba 2: "Cambia el puerto del microservicio de reportes porque choca con mi base de datos."**

* **Solución:** Vas a `ms-analytics-report`, `src/main/resources/application.properties`. Cambias `server.port=8099` a `server.port=8105`. Reinicias.
* **Defensa:** *"Listo, profesor. Como los microservicios son arquitectónicamente independientes, cambiar el puerto en el `application.properties` es suficiente. Solo debemos asegurar que los servicios clientes (vía `WebClient` o `Feign`) apunten a la nueva URL."*

**Prueba 3: "Quiero que el endpoint de enviar notificaciones pueda ser usado sin token para hacer una prueba desde una web externa."**

* **Solución:** Vas a `ms-notification-log`, paquete `config`, clase `SecurityConfig`. En `authorizeHttpRequests`, agregas: `.requestMatchers("/api/notificaciones/").permitAll()` justo antes de `.anyRequest().authenticated()`.
* **Defensa:** *"Modifiqué la cadena de filtros en el `SecurityFilterChain`. Al colocar `permitAll()`, le indico a Spring Security que omita el paso por el `JwtFilter` para esa ruta específica, dejándola pública."*

**Prueba 4: "Modifica la tabla de Vehículos, agrega el año del auto."**

* **Solución:** NO modifiquen el archivo `V1__init.sql`. Creen un archivo nuevo llamado `V2__agregar_anio.sql` en la carpeta `db/migration`. Dentro escriban: `ALTER TABLE vehiculo ADD COLUMN anio INTEGER;`. Agreguen la variable `anio` en `Vehiculo.java` y en su DTO. Reinicien.
* **Defensa:** *"Para no corromper el historial de migraciones de Flyway y mantener la integridad del Checksum de la versión 1, creé un script secuencial V2 con la instrucción `ALTER TABLE`, que es el estándar de la industria para evolucionar el esquema de BD en producción."*
